# 4.22 面试题：委托 · 事件 · Lambda · 匿名函数

对应复习主题：委托与多播、事件模式、Lambda/匿名方法；MES 中回调、设备消息订阅等场景。

---

## 基础（5 道）

### 1. 委托（delegate）是什么？`Action` / `Func` 和它是什么关系？

**答：**  
- **委托**是**方法签名的类型安全封装**，可指向**静态或实例方法**，支持**多播**（`+=`/`-=` 组合多个目标）。  
- **`Action`**：无返回值，泛型重载表示 0～16 个参数。  
- **`Func<T>`**：最后一型参为**返回值**，如 `Func<int, bool>`。  
- **关系**：二者是 **BCL 预定义委托**，减少自定义 `delegate` 声明。  
- **MES**：异步完成回调常用 `Action`、`Func<Task>` 或自定义 `delegate void DataReceived(byte[] payload)`。

---

### 2. 事件（event）和「公开的委托字段」有什么本质区别？

**答：**  
- **`event`** 封装后，**类外部只能 `+=`/`-=`**，不能直接 **`Invoke`**（除非在声明类内部），避免外部随意触发，符合 **发布-订阅** 边界。  
- 公开 `Delegate` 字段则外部可**随意赋值/调用**，破坏封装。  
- **编译器**会为事件生成 `add`/`remove` 访问器（类似属性），背后仍多为 **MulticastDelegate**。

---

### 3. Lambda 表达式和匿名方法有什么区别？什么时候用哪个？

**答：**  
- **匿名方法**（C# 2）：`delegate(int x) { return x * 2; }`，语法冗长。  
- **Lambda**（C# 3+）：`(x) => x * 2` 或 `x => x * 2`，更简洁，**类型可推断**。  
- **选用**：现代代码**几乎全用 Lambda**；仅当需要 **`async` 匿名** 或 **语句块** 时用 `=> { }`。  
- **注意**：Lambda **捕获外层变量**时生成**闭包**，生命周期可能延长到堆上（见进阶）。

---

### 4. 请举一个 MES / 上位机里「用委托或事件做设备消息回调」的例子（口述即可）。

**答：**  
- **MQTT 客户端**：库提供 `MessageReceived` 或 `ApplicationMessageReceivedAsync`，订阅后在回调里解析 **Topic/Payload**，写入**环形缓冲区**或 **推送到 UI 线程**（`Dispatcher`）。  
- **Modbus / 串口**：轮询或异步读完成后，用 **回调/事件** 把寄存器值交给**业务解析层**，避免在驱动里写业务逻辑。  
- 要点：**驱动层只抛数据**，**业务层**订阅处理，解耦测试与替换设备。

---

### 5. 多播委托执行顺序和异常行为是怎样的？

**答：**  
- **`+=` 顺序**通常决定 **调用顺序**（同线程内顺序调用各目标）。  
- 若某一目标**抛异常**，**后续目标是否执行**取决于调用方是否捕获；`.NET` 里 `Delegate.GetInvocationList()` **手动循环**可控制**吞异常/继续**。  
- **事件**：订阅者异常可能导致**其他订阅者收不到通知**，生产代码常在 **事件处理器内 try-catch 或统一调度**。

## 进阶（5 道）

### J1. 为什么说「事件未取消订阅」容易导致内存泄漏？WPF / 长寿命服务里如何避免？

**答：**  
- **订阅者**通过委托**被发布者强引用**，若发布者**长寿命**而订阅者本应释放，则订阅者**无法被 GC**。  
- **避免**：页面/窗口 **`Unloaded`/`Closed` 时 `-=`**；或使用 **`WeakEventManager`**（WPF）；或服务端用 **接口 + 弱引用** 模式。  
- **MES**：**单例 MQTT 客户端** + 多个界面若反复订阅，必须**成对取消**。

---

### J2. `Delegate` 的协变/逆变在哪些场景有用？`Func`/`Action` 为何能这样设计？

**答：**  
- **协变（out）**：返回值可替换为**更派生**类型，如 `Func<Dog>` 可当作 `Func<Animal>` 使用（`Dog : Animal`）。  
- **逆变（in）**：参数可接受**更基类**，如 `Action<Animal>` 可赋值给需要 `Action<Dog>` 的位置（参数方向相反）。  
- **面试**：`Func<out TResult>`、`Action<in T>` 的 `in`/`out` 与**方法组转换**有关，答出「**委托泛型参数有 in/out 标注**」即可。

---

### J3. Lambda 捕获循环变量 `for (int i=0; ...)` 时经典坑是什么？C# 5 以后有何变化？

**答：**  
- **经典坑**：旧版语言中，多个 Lambda **共享同一循环变量**，循环结束后**全部为最终值**。  
- **C# 5+**：`foreach` 的迭代变量**每次循环语义正确**；`for` 仍建议在循环内 **`int local = i` 再捕获** 或重构，避免歧义。  
- **工业**：批量注册设备回调时，**索引错位**会导致**写到错误工位**。

---

### J4. 表达式树（Expression<Func<...>>）和普通 `Func` 委托有何不同？ORM 里为何用它？

**答：**  
- **`Func`**：编译成 **IL，运行时直接执行**。  
- **`Expression<Func<>>`**：编译成 **数据结构（树）**，可**分析、翻译**（如翻译成 **SQL**）。  
- **EF Core**：`Where(x => x.Id > 1)` 通过**表达式树**解析为 **SQL WHERE**，若用 **`Func` 委托**先 `Compile` 再过滤则变成**内存过滤**，易错。  
- **MES**：报表动态条件、规则引擎也可能用表达式树做**可配置逻辑**（进阶话题）。

---

### J5. `async void` 事件处理程序为何危险？工业客户端里推荐怎么写？

**答：**  
- **`async void`**：异常**无法被上层 await 捕获**，易导致**未观察异常**、进程级崩溃难排查。  
- **仅适合** UI 事件入口等框架约定场景，且应在内部 **try-catch**。  
- **推荐**：事件里 **`async Task Handler(...)`** 不成立时，用 **`async void` + 全包 try-catch**，或 **`_ = RunAsync()`** 且内部 **`Task` 观测异常**（如写日志）。  
- **服务**：尽量 **`async Task`** 一路到底。
